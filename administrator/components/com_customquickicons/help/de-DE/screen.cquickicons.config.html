Custom QuickIcons - Übersicht

Aktuellste Version hier
CQI besteht aus 2 Teilen: dieser Komponente hier und einem Modul.
Das Modul kann an 2 Positionen veröffentlicht werden:
  1. icon (ersetzt damit das bisherige Modul quickicons und ist Standardvorgabe)
  2. cpanel (ist dann sichtbar mit weiteren Modulen rechts in der Modulleiste)
Eine weitere Möglichkeit wäre, das Modul zu kopieren und an beiden Positionen zu veröffentlichen, allerdings mit unterschiedlichen Icons.

Menütabs
Die Einstellungen werden in den folgenden Menüpunkten vorgenommen
  • Generell
    • Ziel
      Hier für interne Komponenten den relativen Link (ab index2.php) zum Komponentenaufruf angeben - er wird dann automatisch durch das System ergänzt.
      Werden externe Verknüpfungen angegeben, sind die Links immer in der Form http://www.website_link.com anzugeben!
      anzugeben!
    • Neues Fenster
      Soll die Verknüpfung in einem neuen Fenster aufgemacht werden
    • Aktiviert
      Ist dieses Icon/Menü verfügbar?
    • Reihenfolge
      Bei einer Neuanlage werden neue Menüpunkte automatisch an den Anfang gereiht (Feld ist nicht anklickbar!).
      Eine Umreihung kann erst nach dem Speichern vorgenommen werden
    • Berechtigt
      Hier definieren welche Benutzergruppen Zugriff auf diesen Menüpunkt haben dürfen/sollen - die Berechtigung kann jederzeit geändert werden.
      Berücksichtigt werden nur diejenigen Gruppen welche Zugang zum Backend haben.
      Hinweis: Nach der Komponenteninstallierung werden alle Menüpunkte mit Superadminrechten versehen, das heisst dass alle Menüpunkte vorerst nur vom Superadmin gesehen und aufgerufen werden können
    • Benutzer
      Hier kann dieser Menüpunkt einem (1) bestimmten Benutzer aus den Administratorengruppen zugewiesen werden, das Icon ist dann nur für diesen Benutzer sichtbar
      Hinweis: Diese Art der Zuweisung hat Vorrang vor der Gruppendefinierung!
  • Text
    • Präfix
      Vorzeichen zum Menüpunkttext, kann entweder mit den angeführten Formatierungmöglichkeiten vorgenommen werden oder eigene Vorzeichen definieren
    • Text
      Der eigentliche Menütext welcher später in der Übersicht angezeigt wird. Je kürzer desto besser - weiterer Text welcher angezeigt wird, kann unter Titel angegeben werden (als alt. bzw. title.tag)
    • Vorgebenene Formatierungsmöglichkeiten zur Menüpunktanzeige
      Fett, Schräg, Unterstrichen
      Es kann jede beliebige HTML-Formatierung in beliebiger Kombination eingesetzt werden
    • Postfix
      Der Formatierungsabschluss zum Präfix.
      Hinweis: Falsche Angaben können bis zu einer Nichtanzeige einzelner Menüpunkte gehen!
    • Tastenkürzel
      Für Benutzer mit Beeinträchtigungen und weil es das Gesetz innerhab der EU vorschreibt, sollten alle (bzw. die wichtigsten Menüpunkte) ein Tastaturkürzel besitzen um nicht auf eine Eingabe mit der Maus angewiesen zu sein
      Die Shortcuts werden mittles der Taste ALT + PLUS 1 Zeichen aufgerufen
      Die Kürzel müssen einmalig sein und dürfen nicht 2x für verschiedene Menüs vergeben werden!
    • Titel
      Text welcher angezeigt wird wenn mit der Maus über den Menüpunkt gefahren wird. Hier kann eventuell ein ausführlicher Text stehen, sollte aber nicht länger als 60 Zeichen sein
  • Anzeige
    • Ausgabe als
      Hier definieren wie der Menüpunkt angezeigt werden soll
      • Icon & Text (Vorgabe)
      • Nur Icon
      • Nur Text
    • Bildverzeichnisse
      Hier werden alle innerhalb des CMS gefundenen Bilderordner zru Auswahl angezeigt
    • Icon
      Hier das anzuzeigende Menübild auswählen (es öffnet sich eine neues Fenster)
      Über das Dropdownmenü Bildverzeichnisse kann das Verzeichnis gewählt werden. Es werden alle Verzeichnisse angezeigt welche Bilder enhalten.
      Tipp: über den Mediamanager (siehe Button oben) kann ein eigenes Iconverzeichnis angelegt bzw. ein Vorhandenes aufgerufen und dann diese Icons eingebunden werden
      Hinweis: Wurde der Mediamanager aus CQI aufgerufen um einen neuen Bilderordner zu erstellen, muss CQI selber nochmals über das Menü aufgerufen werden da ansonsten dieser neue Pfad noch nicht in der Dropdownliste enthalten ist!
  • Check
    • Überprüfe Komponente
      Bei internen Verlinkungen kann es sinnvoll sein, vor dem Anzeigen des Menüs zu überprüfen ob diese Komponente noch existiert und nicht eventuell schon deinstalliert wurde!
      Soll eine vorige Überprüfung vorgenommen werden, dann hier mit Ja bestätigen
    • Name & Pfad
      Wenn eine Komponentenüberprüfung stattfinden soll, dann hier den internen Pfad zu dieser Komponente angeben - z.B.: customquickicons/admin.customquickicons.php würde folgenden Pfad ergeben (wird intern ergänzt):
      ../administrator/components/com_customquickicons/admin.customquickicons.php
